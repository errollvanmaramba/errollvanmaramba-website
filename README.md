# Erroll Van Maramba Personal Website

This repository contains the source code for a lightweight, single‑page website that showcases the professional journey of **Erroll Van Maramba**, a marketing designer specializing in AI‑assisted visual storytelling and digital content creation. The site is built using semantic HTML, modern CSS and a small amount of vanilla JavaScript. It is responsive, accessible and ready to deploy.

## Features

- **Hero section** with the name, positioning statement and call‑to‑action.
- **About** section telling the career story with skills, education and certifications.
- **Professional Experience** timeline with roles, dates and achievements.
- **Contact** section with a demo contact form and direct contact links.
- Smooth scrolling navigation and responsive hamburger menu for mobile devices.
- Minimalistic, modern design with a focus on typography, spacing and accessibility.
- Lightweight – no external frameworks or heavy dependencies.

## Running Locally

1. Download or clone this repository.
2. Ensure the directory structure remains intact.
3. Open `index.html` in your preferred web browser.

No build process or server is required; everything runs client‑side.

## Deploying to GitHub Pages

1. Create a new repository on GitHub (e.g., `erroll-van-maramba-website`).
2. Commit and push all files in this project to your new repository.
3. In your repository settings, navigate to **Pages**.
4. Under **Source**, select the branch (usually `main`) and root directory, then click **Save**.
5. GitHub Pages will provide a URL (e.g., `https://yourusername.github.io/erroll-van-maramba-website`). Visit it to see the live site.

For detailed guidance, refer to GitHub’s [Deploying to GitHub Pages documentation](https://docs.github.com/en/pages/getting-started-with-github-pages).

## Customization

You can customize text, colors and other details to suit your needs:

- **Text content**: Open `index.html` and modify the headings, paragraphs, lists and contact information. Placeholder items like `[Add link]` should be replaced with your real GitHub link or removed.
- **Colors and fonts**: Edit the CSS variables at the top of `styles.css` (`--primary-color`, `--secondary-color`, etc.) to adjust the color scheme. You can also change the font stack in the `body` rule.
- **Hero background**: The hero section uses a decorative image located at `assets/hero-bg.png`. Replace this file with your own image (preferably similar dimensions) for a different look.
- **Navigation**: To add or remove navigation links, update the `<nav>` element in `index.html` and corresponding section IDs.

Remember to test your changes across different screen sizes to maintain responsiveness and accessibility.
